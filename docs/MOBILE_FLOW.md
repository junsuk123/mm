# 모바일 설문 흐름

아래 이미지는 `templates/mobile.html`의 실제 상태와 스타일을 문서용 미리보기 데이터로 렌더링한 것입니다. 캡처 과정은 운영 세션이나 참가자 파일을 변경하지 않습니다.

## 1. 첫 접속과 끼니 선택

이름 또는 별명을 입력합니다. 비워 두면 자동 별칭이 생성됩니다. 점심 또는 저녁을 반드시 선택해야 제출할 수 있습니다.

![첫 접속](screenshots/mobile-01-entry.png)

## 2. 최근 음식 선택

첫 화면에서 최근 음식 대분류를 누르면 해당 대분류의 음식 선택 화면으로 자동 이동합니다. 여기서 최근 먹은 음식 한 개를 선택합니다.

![최근 음식 선택](screenshots/mobile-02-recent-food.png)

## 3. 첫 번째 선호 대분류

최근 음식을 고르면 첫 번째 선호 대분류 화면으로 자동 이동합니다.

![첫 번째 선호 대분류](screenshots/mobile-03-preference-1-category.png)

## 4. 첫 번째 선호 음식 두 개

첫 번째 선호 대분류를 고른 뒤 음식 두 개를 선택합니다.

![첫 번째 선호 음식](screenshots/mobile-04-preference-1-foods.png)

## 5. 두 번째 선호 대분류

첫 번째 음식 두 개를 고르면 두 번째 선호 대분류 화면으로 이동합니다. 첫 번째 대분류는 선택할 수 없습니다.

![두 번째 선호 대분류](screenshots/mobile-05-preference-2-category.png)

## 6. 두 번째 선호 음식 두 개

두 번째 선호 대분류에서도 음식 두 개를 선택합니다.

![두 번째 선호 음식](screenshots/mobile-06-preference-2-foods.png)

## 7. 제출 준비

최근 음식과 선호 음식 네 개, 끼니가 모두 선택되면 `제출하기` 버튼이 활성화됩니다.

![제출 준비](screenshots/mobile-07-ready.png)

## 8. 제출 완료와 추천 대기

제출 데이터는 영구 참가자 폴더와 현재 세션에 반영됩니다. 페이지는 2.5초 간격으로 현재 세션 추천을 확인합니다.

![제출 완료](screenshots/mobile-08-submitted.png)

## 9. 이전 추천 평가

기존 사용자가 새 세션에 접속하면 직전 다른 세션의 추천을 먼저 보여줍니다. `안 갈래요`는 사용자 제외 목록에 저장되고 X는 평가 없이 닫습니다.

![이전 추천 평가](screenshots/mobile-09-previous-feedback.png)

## 10. 현재 추천과 길찾기

관리자가 접수를 마감하고 추천 계산을 완료하면 참가자 자신의 그룹 추천이 자동으로 표시됩니다.

위치 길찾기가 전체 활성화 상태이고 아직 동의하지 않았다면 동의 화면을 표시합니다. 좌표는 브라우저에서 네이버 지도 URL을 만드는 데만 사용합니다.

![최종 추천과 길찾기](screenshots/mobile-10-result.png)

## 상태별 동작

| 세션 상태 | 모바일 동작 |
|---|---|
| `collecting` | 설문 입력·제출 가능 |
| `closed` | 제출 비활성화, 추천 계산 대기 메시지 |
| `completed` | 참가자 추천 이력을 불러와 결과 팝업 표시 |
| `failed` | 결과가 생성되지 않으며 관리자에서 CLI 오류 확인 |

## 다시하기

상단 `다시하기`는 `sessionStorage`에 일회성 플래그를 두고 페이지를 새로고침합니다. 다음 로드에서는 저장 프로필을 자동 입력하지 않아 빈 설문으로 시작합니다.
