# 상업적 데이터 제공 준수사항

이 문서는 운영 체크리스트이며 법률 자문이 아닙니다. 실제 제공 전 개인정보·데이터 거래 전문 검토가 필요합니다.

## 현재 프로젝트에서 먼저 해결할 사항

현재 모바일 설문은 음식 추천 서비스 제공을 중심으로 동작하며, 특정 기업에 상업적으로 데이터를 제공한다는 별도 고지·동의 화면은 구현되어 있지 않습니다.

따라서 기존 응답의 참가자 단위 자료를 배민·네이버·쿠팡 등 제3자에게 바로 제공하지 마세요. 우선 집계 데이터만 검토하고, 신규 수집부터 다음 항목을 명확히 고지하는 절차를 설계해야 합니다.

- 제공받는 자 또는 제공받는 자의 범주
- 제공 목적
- 제공 항목
- 보유·이용 기간
- 동의 거부권과 불이익
- 철회·삭제·처리정지 방법

## 개인정보와 가명정보

- 단순히 이름과 기기 UUID를 제거해도 음식 취향, 시간, 상권, 소규모 그룹 정보가 다른 정보와 결합되면 개인을 알아볼 수 있습니다.
- `participants_anonymized.csv`는 자동으로 익명정보라고 단정하지 않고 제한 자료로 관리합니다.
- 통계·과학적 연구 목적의 가명정보 처리 규정이 상업 판매 전반을 자동 허용하는 것은 아닙니다.
- 재식별을 시도하거나 재식별 가능한 추가 정보를 제공해서는 안 됩니다.

## 계약에 포함할 내용

- 데이터와 파생 결과물의 소유·사용권
- 허용 목적과 금지 목적
- 재판매·재제공·재결합·재식별 금지
- 접근 가능한 담당자와 보안 통제
- 보유 기간과 파기 증빙
- 오류 정정과 품질 책임 범위
- 유출 통지와 사고 대응
- 대금, 검수, 업데이트 주기, 계약 종료
- 국외 이전 또는 해외 클라우드 사용 여부

## 보안 운영

- 외부 전달은 기본 `archive` 명령만 사용합니다.
- 운영 `dataset/` 폴더와 제한 CSV를 이메일·메신저로 직접 보내지 않습니다.
- 전달 전 `verify`, 전달 후 `sha256sum -c`를 수행합니다.
- 최소 권한 저장소와 암호화 전송을 사용합니다.
- 수신자, 계약 번호, 전달 파일, 해시, 전달일, 파기일을 기록합니다.
- 유출 사고가 발생하면 적용 법령상 통지·신고 요건과 기한을 즉시 검토합니다.

## 데이터 품질·공정성

- 샘플 10명은 데모이며 상업 데이터에서 제외합니다.
- 자발적 응답은 연령·지역·플랫폼 이용자 전체를 대표하지 않습니다.
- 특정 음식·지역·식당에 불리한 결론을 내릴 때 표본 수와 불확실성을 표시합니다.
- 현재 추천 점수가 인원수 가중치를 사용하지 않는다는 점을 수신자에게 알립니다.
