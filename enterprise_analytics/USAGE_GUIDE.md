# 데이터 활용성과 사용 방법

## 플랫폼별 활용 가능성

아래는 배달·지도·커머스 요식업 사업자와 협의할 수 있는 분석 예시이며, 특정 기업과의 제휴나 구매를 보장하지 않습니다.

### 배달 플랫폼

- 시간대·끼니별 음식 선호 군집
- 함께 주문할 가능성이 높은 음식 카테고리 조합
- 최근 섭취 음식과 다른 메뉴를 제안하는 다양성 추천
- 그룹 주문용 메뉴 큐레이션
- 특정 상권의 신규 입점 카테고리 가설

### 지도·지역 검색 플랫폼

- 검색 위치별 선호 카테고리와 예상 이동 허용 범위
- 리뷰 순위·거리 필터와 최종 선택 간 관계
- 외식 그룹의 공통 취향 기반 장소 추천
- 추천 이후 `안 갈래요` 피드백을 이용한 품질 개선

### 커머스·퀵커머스

- 음식 선호와 간편식·밀키트 카테고리 수요 가설
- 점심·저녁별 상품 구성과 프로모션 세분화
- 교차 카테고리 번들 및 지역별 재고 실험 설계

## 권장 분석 단위

외부 제공의 기본 단위는 개인 행이 아니라 다음 집계입니다.

- 세션별 카테고리 선택 빈도
- 그룹별 공통 선호 빈도
- 그룹별 추천 식당·카테고리
- 거리·도보·리뷰 필터 조건
- 추천 제외 식당 수

## 사용 절차

1. `verify`로 패키지를 검사합니다.
2. 데모 참가자를 제외했는지 확인합니다.
3. 모집 방식, 기간, 상권, 표본 수를 데이터 설명서에 기록합니다.
4. 외부 제공 후보 파일만 `archive`합니다.
5. 수신 업체와 목적·허용 분석·보유기간·재제공 금지를 계약합니다.
6. SHA-256과 전달·삭제 이력을 보관합니다.
7. 결과 보고서에는 표본 편향과 점수 규칙을 함께 표시합니다.

## 권장 KPI

- 유효 모바일 응답 수
- 반복 참여율
- 카테고리별 선택률
- 그룹 내 공통 선호율
- 추천 결과의 거리·도보 분포
- 추천 이후 제외 피드백률
- 세션별 추천 성공률

## 금지 또는 주의가 필요한 활용

- 개인별 타깃 광고나 개인 프로파일링
- 운영 ID 또는 다른 데이터와의 재결합
- 소수 그룹·희귀 취향을 이용한 개인 추정
- 참여자가 예상하지 못한 상업적 제3자 제공
- 데이터가 대표 표본인 것처럼 과장하는 시장 규모 추정
